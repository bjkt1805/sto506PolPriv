¿Qué es la accesibilidad web?

Se conoce como Accesibilidad Web a la disponibilidad de una página web para el máximo número de personas. Esto quiere decir, el sitio web debe ser accesible para cualquier persona, independientemente de su edad, capacidades técnicas, equipo final o limitaciones físicas (visuales, cognitivas, etc.). En otras palabras, accesibilidad es sinónimo de facilidad en el acceso para el usuario final.

Importancia de la accesibilidad web

Dado a que existen personas que sufren discapacidades motoras, físicas (visuales, auditivas) y/o cognitivas que pueden representar limitaciones a la hora de acceder a un sitio web, es de suma importancia que este posea herramientas de facilidad en su uso tanto para estas personas discapacitadas como los usuarios sin ninguna limitación. Esto garantiza un uso efectivo del sitio web para cualquier usuario (objetivo último de la accesibilidad web)

Principios

Principios de accesibilidad web
Principio Definición
Perceptible La interfaz y los contenidos del sitio web deben ser distinguibles por cualquier usuario. La inserción de imágenes, videos, audio o cualquier elemento multimedia ayuda a las personas con discapacidad a percibir el contenido de la página fácilmente. Un diseño responsivo es de suma importancia ya que el sitio web debe adaptarse a diferentes formas, pantallas, zooms elevados, etc. La distinción entre el primer plano y el fondo es imperante y los diseñadores son responsables de aplicar escalas de contrastes y colores que adapten a todos los usuarios.
Operable Cualquier usuario con acceso a la página web debe poder interactuar con la interfaz del sitio sin ningú problema. Por ejemplo, si los usuarios no tienen mouse deben poder interactuar con el sitio por medio del teclado. El sitio debe ser navegable fácilmente.
Comprensible La información que contiene el sitio debe ser entendible y evaluable/interpretable. El contenido debe ser no ambiguo, claro, conciso y que proporcione una guía. Esto último significa que aquellos elementos del sitio que colectan información del usuario deben proveer una pequeña guía de la información que se colecta, cómo se inserta y para qué fin se va a utiliza esa información.
Robusto El sitio web debe estar bien estructurado y poseer una seguridad sólida. Además debe incluir tecnologías de apoyo para las personas con discapacidad. Por ejemplo: lupas amplificadoras o lectores de voz alta adoptan el contenido para que sea consumida por personas discapacitadas.

Ejemplos

  • Perceptible: La página web de Scope (organización benéfica de igualdad y discapacidad ubicada en Gales e Inglaterra) ofrece un diseño perceptible para cualquier usuario. Posee letras grandes, contrastes altos y ofrece personalización del sitio para que los usuarios puedan adaptar el sitio web según sus capacidades visuales, auditivas, etc.
  • Operable: El sitio web de BBC iPlayer ofrece una interfaz amigable con el usuario, donde predominan los contrastes fuertes. También ofrece una navegabilidad intuitiva que permite a cualquier usuario operar el reproductor multimedia de forma sencilla.
  • Comprensible: La página oficial del gobierno del Reino Unido ofrece de primera vista el objetivo principal de su uso (información y servicios del gobierno). Además, carga en su inicio una lista de servicios populares que ayudan a cualquier persona acceder a ellos fácilmente y con poco esfuerzo.
  • Robusto: El sitio web de la Autoridad de Transporte Metropolitano de Nueva York además de poseer una interfaz amigable y accesible para el usuario, posee un diseño bastante robusto que permite al usuario comprender de manera fácil todos los servicios ofrecidos por el servicio de transporte.
